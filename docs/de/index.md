# Kizuna

Kizuna verbindet Projektarbeit in GitHub Issues mit persönlichen Produktivitätssystemen wie Kalendern und Erinnerungen. Das Projekt gehört als eigenständige Software zum Dokoni-Ökosystem.

## Ziel

Kizuna erprobt wartbare Integrationen, die Informationen zwischen Projektarbeit und persönlichen Arbeitsabläufen nutzbar machen. Die Entwicklung erfolgt in kleinen Mocks, damit jede neue Fähigkeit separat verstanden und geprüft werden kann.

## Entwicklungsmodell

- Mock 1 und Mock 2 sind abgeschlossen.
- Mock 2 hat die modulare Grundlage geschaffen.
- Mock 3 baut auf dieser Architektur auf und konzentriert sich auf das Erstellen und Aktualisieren von GitHub Issues.
- Änderungen sollen inkrementell und testgetrieben erfolgen.

Kizuna ist eine Anwendung und keine Spezifikation. AMTS kann zur Organisation des Projektwissens verwendet werden, bleibt aber ein unabhängiges Projekt.
