# Status and roadmap

## Current state

Kizuna is in active development. Mock 3 explores write access to GitHub Issues and is intended to reuse the existing Mock 2 architecture.

## Next steps

1. Select the target repository and define its configuration.
2. Design a clearly separated GitHub write layer.
3. Implement issue creation.
4. Prepare issue updates.
5. Add tests and public documentation.

## Not yet defined

Supported calendar and reminder systems, the eventual distribution format, and a release schedule have not been committed. Reliable details will be published when the relevant development stage is complete.
