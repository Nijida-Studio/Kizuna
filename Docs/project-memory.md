# Kizuna Project Memory

Kizuna is part of the Dokoni Ecosystem.

## Development approach

Kizuna is currently developed through small CLI mocks before the main development phase.

The mocks are intentionally simple and are used to test structure, principles and implementation patterns.

## Completed work

### Mock 2

Mock 2 implemented loading and filtering of issue-like data.

Implemented capabilities:

- Swift Package structure
- CLI application target
- KizunaCore library target
- issue models
- JSON issue loading
- issue filtering by type, status, assigned user and subtype
- basic app directory/environment structure
- tests for core initialization and issue filtering
- documentation in Docs/mock2.md
