## Current focus

Mock 3 explores creating and updating GitHub Issues. The modular Mock 2 structure should remain intact and only be extended where the new capability requires it.

## Next steps

2. Design the GitHub write layer.
3. Implement issue creation.
4. Prepare issue updates.
5. Add tests and documentation.

Calendar and reminder systems, distribution, and schedule will only be announced after reliable decisions have been made.
