# Kizuna

Kizuna connects project work in GitHub Issues with personal productivity systems such as calendars and reminders. It is an independent software project within the Dokoni ecosystem.

## Purpose

Kizuna explores maintainable integrations that make project information useful in personal workflows. Development is divided into small mocks so each new capability can be understood and tested independently.

## Development model

- Mock 1 and Mock 2 are complete.
- Mock 2 established the modular foundation.
- Mock 3 builds on that architecture and focuses on creating and updating GitHub Issues.
- Changes should remain incremental and test-driven.

Kizuna is an application, not a specification. AMTS may organize its project knowledge but remains an independent project.
