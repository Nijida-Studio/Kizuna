## Development

- Prefer extending the existing architecture to broad redesigns.
- Explore new capabilities in clearly scoped mocks.
- Implement and test changes incrementally.
- Maintain public documentation alongside implementation.

## Project boundaries

Kizuna is an independent software project. AMTS may organize its project knowledge but remains a separate specification and defines neither the product nor its architecture.
